## CRUD Pattern (read this before implementing a new entity)

Every entity follows the same 7-file pattern. Use Faction as the reference (simplest case).

**Files per entity (example: Faction → replace with new entity name):**
- `schemas/faction.ts` — Zod: createSchema + updateSchema + idSchema, extends baseEntityInputSchema
- `server/faction-service.ts` — list / get / create / update / delete; soft-delete cleans EntityLink + EntityTag
- `server/slug.ts` — add resolveFactionSlug() using uniqueSlug(prisma.faction, ...)
- `app/api/factions/route.ts` — GET + POST
- `app/api/factions/[id]/route.ts` — GET + PATCH + DELETE
- `features/factions/faction-form.tsx` — "use client" form; reads ui.factions.form.*
- `app/factions/` — page.tsx, new/page.tsx, [id]/page.tsx, [id]/edit/page.tsx, loading.tsx, error.tsx
- `lib/i18n/ui.ts` — add nav key + entity block (ru + en); Russian strings must be real Cyrillic, no copy-paste from terminal
- `components/site-nav.tsx` — add nav link

**Key conventions:**
- Primary key field: `name` (Character, Place, Faction) or `title` (Story, Event, LoreEntry, RuleSystem)
- entityType strings used in EntityTag/EntityLink: `'character'`, `'place'`, `'faction'`, `'lore-entry'`, `'story'`, `'event'`
- TagManager and EntityLinkManager props: extend the `entityType` union in both components when adding a new entity
- Route params: `context.params` is a Promise in Next.js App Router — always `await context.params`
- JSON fields: use `toJsonWrite(existing.field)` in updates to avoid NullableJsonNullValueInput type error
- baseEntityInputSchema already covers: slug, summary, content, status, canonState, metadata

**Relation widgets (not full CRUD pages):** PlaceConnectionManager, CharacterRelationManager, StoryEntityManager, TagManager, EntityLinkManager
- Pattern: service (list/create/delete) + POST+DELETE API routes + "use client" panel widget wired into detail page
- Widget receives pre-fetched lists as props; names/hrefs resolved server-side before passing down
- entityType strings in EntityTag/EntityLink: 'character', 'place', 'faction', 'lore-entry', 'rule-system'
