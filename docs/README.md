# Docs Notes

These notes mix two kinds of material:

- current-foundation documents that describe the real active app direction
- forward-looking domain documents that preserve useful structure for later phases

When in doubt:
- trust the runtime code for what is currently implemented
- use the docs to understand why the broader data model exists
- do not assume every documented entity or system has active UI yet

Suggested reading order:
- `mvp-scope.md`
- `repo-structure.md`
- `domain-model.md`
- `map-model.md`
